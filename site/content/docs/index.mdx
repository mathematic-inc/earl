Earl is a CLI and MCP server that connects AI agents to external services — HTTP, GraphQL, gRPC, Bash, SQL — while keeping secrets, write operations, and network egress under human control.

## The problem it solves

In early 2026, OpenClaw — the dominant open-source AI agent — had a rough stretch: 341 malicious skills on its ClawHub marketplace, a 1-click RCE CVE, command injection in SSH, credential exfiltration through prompt injection. Different vectors, same root cause.

ClawHub skills are Markdown documents that the LLM reads and executes. Prompt injection isn't an edge case in that model — it's baked in. No amount of review process changes that.

Earl takes a different approach. Operations are HCL files committed to your repository. The LLM calls a template by name; it never reads the template content. Credentials stay in the OS keychain, never in LLM context. Write operations require explicit authorization. The network layer blocks private and loopback addresses with no config bypass.

More detail on each of these: [How Earl Works](/docs/how-earl-works).
