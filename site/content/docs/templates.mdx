Templates are HCL files that define what an AI agent is allowed to do with a provider. Every command an agent can invoke maps to exactly one `command` block. The LLM sees the command's title, summary, description, and parameter names. It never reads the template body — the URL, method, headers, auth setup, and request structure. That separation means the agent cannot alter the operation itself, only supply the parameter values the template author declared.

## Mental model

A template is a typed interface between an AI agent and an external service. You, the template author, write the full operation: URL, method, headers, auth. The agent supplies only the parameter values you declare. Nothing else reaches the wire.

At call time, Earl runs two steps in order:

1. Parse HCL into a structured document
2. Render Jinja expressions against `args` (the resolved parameters) and any environment variables

The order is what trips people up. HCL is parsed first, before Jinja runs. That means **every `{{ }}` expression must live inside a valid HCL string value**:

```hcl
# WRONG — HCL fails to parse this before Jinja ever sees it
params = [{{ args.limit }}]

# CORRECT — Jinja expression inside an HCL string
params = ["{{ args.limit }}"]
```

When a string is nothing but a Jinja expression — `"{{ args.limit }}"` with no surrounding text — Earl coerces the rendered result to the correct JSON type. An integer stays an integer. No quotes leak through.

## File locations

| Scope           | Path                                      |
| --------------- | ----------------------------------------- |
| Local (project) | `./templates/<provider>.hcl`              |
| Global          | `~/.config/earl/templates/<provider>.hcl` |

A local template with the same provider name shadows the global one.

## File structure

A minimal template:

```hcl
version    = 1
provider   = "github"
categories = ["scm", "issues"]

command "list_repos" {
  title       = "List repositories"
  summary     = "List repos for the authenticated user"
  description = "Fetch repositories for the authenticated user."

  annotations {
    mode    = "read"
    secrets = ["github.token"]
  }

  param "per_page" {
    type        = "integer"
    required    = false
    default     = 30
    description = "Results per page (max 100)"
  }

  operation {
    protocol = "http"
    method   = "GET"
    url      = "https://api.github.com/user/repos"

    auth {
      kind   = "bearer"
      secret = "github.token"
    }

    query = {
      per_page = "{{ args.per_page }}"
    }

    headers = {
      Accept               = "application/vnd.github+json"
      X-GitHub-Api-Version = "2022-11-28"
    }
  }

  result {
    decode = "json"
    output = "Found {{ result | length }} repos:\n{% for r in result %}  - {{ r.full_name }}\n{% endfor %}"
  }
}
```

### Top-level fields

| Field              | Required | Description                                                                   |
| ------------------ | -------- | ----------------------------------------------------------------------------- |
| `version`          | yes      | Schema version. Currently `1`.                                                |
| `provider`         | yes      | Identifier for calling commands: `earl call github.list_repos`.               |
| `categories`       | no       | Labels for discovery and filtering.                                           |
| `environments`     | no       | Provider-level environment switching. See [Environments](/docs/environments). |
| `command "<name>"` | yes      | One or more command blocks.                                                   |

### Command fields

| Field                   | Required | Description                                                                   |
| ----------------------- | -------- | ----------------------------------------------------------------------------- |
| `title`                 | yes      | Short display name shown in listings.                                         |
| `summary`               | yes      | One-line description shown to the agent.                                      |
| `description`           | yes      | Full description. The agent reads this to decide whether to call the command. |
| `categories`            | no       | Labels for this specific command.                                             |
| `annotations`           | no       | Mode, secrets, and environment switching flags.                               |
| `param "<name>"`        | no       | Repeatable. Defines a parameter the agent can supply.                         |
| `operation`             | yes      | The request to execute.                                                       |
| `result`                | no       | How to decode and format the response.                                        |
| `environment_overrides` | no       | Per-environment operation replacements.                                       |

## Annotations

```hcl
annotations {
  mode    = "read"   # or "write"
  secrets = ["github.token"]
  allow_environment_protocol_switching = false
}
```

`mode = "write"` is the default. Earl prompts for confirmation unless the caller passes `--yes`. Set `mode = "read"` explicitly for commands that only retrieve data — omitting the annotations block means the command will prompt.

`secrets` lists which secrets the command needs. Earl checks that these exist before running.

## Parameters

```hcl
param "query" {
  type        = "string"
  required    = true
  description = "Search query (e.g. 'language:rust stars:>100')"
}

param "per_page" {
  type        = "integer"
  required    = false
  default     = 20
  description = "Results per page (max 100)"
}
```

Supported types: `string`, `integer`, `number`, `boolean`, `null`, `array`, `object`.

Inside the `operation` block, parameters are available as `args.<name>`:

```hcl
url = "https://api.github.com/repos/{{ args.owner }}/{{ args.repo }}"
```

## The six protocols

### HTTP

```hcl
operation {
  protocol = "http"
  method   = "GET"   # GET, POST, PUT, PATCH, DELETE
  url      = "https://api.example.com/items/{{ args.id }}"
  stream   = false

  auth {
    kind   = "bearer"
    secret = "provider.token"
  }

  headers = { Accept = "application/json" }

  query = { filter = "{{ args.filter }}" }

  body {
    kind  = "json"   # json, form_urlencoded, raw_text, raw_bytes_base64, multipart, file_stream
    value = {
      name  = "{{ args.name }}"
      count = "{{ args.count }}"
    }
  }
}
```

HTTP fields sit at the top level of `operation`. There is no nested `http { }` wrapper.

### GraphQL

```hcl
operation {
  protocol = "graphql"
  url      = "https://api.github.com/graphql"

  auth {
    kind   = "bearer"
    secret = "github.token"
  }

  graphql {
    query = <<-GQL
      query($owner: String!, $repo: String!) {
        repository(owner: $owner, name: $repo) {
          stargazerCount
          description
        }
      }
    GQL
    variables = {
      owner = "{{ args.owner }}"
      repo  = "{{ args.repo }}"
    }
  }
}
```

The query goes in a heredoc. Variables go in `graphql { variables = { ... } }`.

### gRPC

```hcl
operation {
  protocol = "grpc"
  url      = "https://grpc.example.com"

  grpc {
    service             = "example.v1.ExampleService"
    method              = "GetItem"
    descriptor_set_file = "example.pb"   # optional; omit to use server reflection
    body = {
      id = "{{ args.id }}"
    }
  }
}
```

Earl uses gRPC reflection v1. If the server only supports v1alpha or doesn't expose reflection at all, provide a compiled descriptor set via `descriptor_set_file`. See [Streaming](/docs/streaming) for server-streaming RPCs.

### Bash

```hcl
operation {
  protocol = "bash"

  bash {
    script = <<-SH
      jq -r '.[] | .name' {{ args.input_file }}
    SH
    sandbox {
      network          = false
      max_time_ms      = 30000
      max_output_bytes = 1048576
    }
  }
}
```

Bash commands run in a sandbox with network off by default. Earl also blocks all private and loopback IP ranges at the SSRF layer — sandbox settings don't change that.

### SQL

```hcl
operation {
  protocol = "sql"

  sql {
    connection_secret = "myapp.db_url"
    query             = "SELECT id, name FROM users WHERE status = $1 LIMIT $2"
    params            = ["{{ args.status }}", "{{ args.limit }}"]
    sandbox {
      read_only = true
      max_rows  = 100
    }
  }
}
```

SQL parameters go through the driver's parameter interface, not string interpolation. Quote every Jinja expression in the array — HCL needs them as strings, but Earl coerces them to the right types before sending the query. Placeholder syntax is database-specific: `$1`, `$2`... for PostgreSQL; `?` for MySQL and SQLite.

### Browser

```hcl
operation {
  protocol = "browser"

  browser {
    session_id            = "{{ args.session_id }}"  # optional; omit for one-shot
    headless              = true
    timeout_ms            = 30000
    on_failure_screenshot = true
    steps = [
      { action = "navigate",  url      = "{{ args.url }}" },
      { action = "fill",      selector = "#username", text = "{{ args.user }}" },
      { action = "fill",      selector = "#password", text = "{{ secrets['site.password'] }}" },
      { action = "click",     selector = "button[type=submit]" },
      { action = "wait_for",  text = "Dashboard", timeout_ms = 5000 },
      { action = "snapshot" },
    ]
  }
}
```

Each command runs a sequence of `steps`. Every step is an inline object with an `action` field and zero or more action-specific fields. The result of the command is the last step's output — a JSON object whose shape depends on the action (e.g. `snapshot` returns `{"text": "..."}`, `evaluate` returns `{"value": ...}`).

Steps support two cross-cutting fields available on every action:

| Field        | Default | Description                                                           |
| ------------ | ------- | --------------------------------------------------------------------- |
| `optional`   | `false` | When `true`, a failure on this step is skipped and execution continues. |
| `timeout_ms` | —       | Per-step timeout, overrides the command-level `timeout_ms`.           |

**Session persistence** — set `session_id` to a stable string to keep the browser alive between commands. A subsequent command with the same `session_id` reconnects to the same browser instance, preserving cookies, localStorage, and page state. Omit `session_id` for one-shot commands that start and stop a fresh browser.

The `snapshot` action returns an accessibility tree as a text description. Use the `ref` handles it contains to identify elements for subsequent `click`, `fill`, and other interaction steps without needing CSS selectors.

For the full list of actions and their fields, see [Template Schema — Browser steps](/docs/template-schema#step-reference).

## Auth

```hcl
# Bearer token
auth {
  kind   = "bearer"
  secret = "provider.token"
}

# API key in a header
auth {
  kind     = "api_key"
  secret   = "provider.api_key"
  location = "header"    # header, query, cookie
  name     = "X-Api-Key"
}

# HTTP Basic
auth {
  kind            = "basic"
  username        = "myuser"
  password_secret = "provider.password"
}

# OAuth2 (references a profile in config.toml)
auth {
  kind    = "o_auth2_profile"
  profile = "myprofile"
}
```

See [Secrets and Auth](/docs/secrets-and-auth) for configuration details.

## Result block

```hcl
result {
  decode = "json"   # auto, json, text, html, xml, binary
  output = "Found {{ result.total_count }} items:\n{% for item in result.items[:10] %}  - {{ item.name }}\n{% endfor %}"
}
```

`decode` controls how response bytes are parsed before the output template runs:

- `auto` — infer from `Content-Type`, then fall back to JSON detection (default)
- `json` — parse as JSON; `result` is the decoded object
- `text` — `result` is the raw string
- `html` — treat as HTML; `result` is a string
- `xml` — treat as XML; `result` is a string
- `binary` — return raw bytes base64-encoded as a string

For streaming, the decode mode applies per chunk. See [Streaming](/docs/streaming).

## Environment overrides

When the active environment matches, the override replaces the command's default operation entirely:

```hcl
environment_overrides {
  staging {
    operation {
      protocol = "http"
      method   = "GET"
      url      = "https://staging.github.internal/search/repositories"
      auth {
        kind   = "bearer"
        secret = "github.token"
      }
    }
  }
}
```

See [Environments](/docs/environments) for how environments are configured and activated.

## Complete example

```hcl
version    = 1
provider   = "github"
categories = ["scm", "issues"]

environments {
  default = "production"
  secrets = ["github.token"]
  production {
    base_url = "https://api.github.com"
  }
  staging {
    base_url = "https://staging.github.internal"
  }
}

command "search_repos" {
  title       = "Search repositories"
  summary     = "Search GitHub repos by query"
  description = <<-EOT
    Search GitHub repositories using GitHub's search API.
    Supports qualifiers like language:rust, stars:>100, topic:cli.

    ## Guidance for AI agents
    Use this command to find repositories by topic, language, or popularity.
  EOT
  categories  = ["search", "repos"]

  annotations {
    mode    = "read"
    secrets = ["github.token"]
  }

  param "query" {
    type        = "string"
    required    = true
    description = "Search query (e.g. 'language:rust stars:>100 topic:cli')"
  }

  param "per_page" {
    type        = "integer"
    required    = false
    default     = 20
    description = "Results per page (max 100)"
  }

  operation {
    protocol = "http"
    method   = "GET"
    url      = "https://api.github.com/search/repositories"

    auth {
      kind   = "bearer"
      secret = "github.token"
    }

    headers = {
      Accept               = "application/vnd.github+json"
      X-GitHub-Api-Version = "2022-11-28"
    }

    query = {
      q        = "{{ args.query }}"
      per_page = "{{ args.per_page }}"
      order    = "desc"
    }
  }

  result {
    decode = "json"
    output = "Found {{ result.total_count }} repos:\n{% for r in result.items[:5] %}  - {{ r.full_name }} (★{{ r.stargazers_count }}) — {{ r.description | default('') }}\n{% endfor %}"
  }
}

command "create_issue" {
  title       = "Create issue"
  summary     = "Create a new issue in a repository"
  description = "Create a GitHub issue with a title and optional body."
  categories  = ["issues"]

  annotations {
    mode    = "write"
    secrets = ["github.token"]
  }

  param "owner" {
    type        = "string"
    required    = true
    description = "Repository owner"
  }

  param "repo" {
    type        = "string"
    required    = true
    description = "Repository name"
  }

  param "title" {
    type        = "string"
    required    = true
    description = "Issue title"
  }

  param "body" {
    type        = "string"
    required    = false
    default     = ""
    description = "Issue body (Markdown)"
  }

  operation {
    protocol = "http"
    method   = "POST"
    url      = "https://api.github.com/repos/{{ args.owner }}/{{ args.repo }}/issues"

    auth {
      kind   = "bearer"
      secret = "github.token"
    }

    headers = {
      Accept               = "application/vnd.github+json"
      X-GitHub-Api-Version = "2022-11-28"
    }

    body {
      kind  = "json"
      value = {
        title = "{{ args.title }}"
        body  = "{{ args.body }}"
      }
    }
  }

  result {
    decode = "json"
    output = "Created issue #{{ result.number }}: {{ result.title }}\nURL: {{ result.html_url }}"
  }
}
```

For the complete field-by-field reference, see [Template Schema](/docs/template-schema).
